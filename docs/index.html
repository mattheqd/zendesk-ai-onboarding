Getting Started

Don't skip this page.

Claude Code runs locally on your machine. Getting it set up properly takes a few prerequisites. Ten minutes now saves hours later. Skip nothing.

What you're installing and why:

Claude Code is not a website you log into. It's an agent that runs on your laptop, talks to your filesystem, and edits real files, authenticated to Zendesk's internal AI Gateway. Setting that up takes a few small installs. Do it once and you have these superpowers forever.

Before you run setup

1. Connect to GlobalProtect VPN

The AI Gateway and all internal Zendesk services are behind the corporate network. If GlobalProtect is off, the setup will fail. Connect to VPN before running the command below.

2. Get GitHub access

If you're not already in the zendesk GitHub org, you need access to install internal plugins and access Zendesk repositories. Request access as early as possible—it can take time depending on IT's queue.

Action: Request access via Okta Zendesk Hub:

  1. Log into Okta and open Zendesk Hub
  2. Navigate to Productiv
  3. Request access to the Zendesk GitHub organization
  4. Include your GitHub username in the request

Quick Setup

Once VPN is connected, open Terminal (Cmd+Space, type "Terminal", hit Return) and paste:

bash <(curl -fsSL https://raw.githubusercontent.com/zendesk/designer-ai-onboarding/main/setup-wizard.sh)

The wizard will guide you through the rest.

Note: You may be asked for your Mac password during setup. When you type it, nothing will appear on screen (not even dots) — this is normal security behavior. Just type your password and press Enter.

Common Issues

Troubleshooting

"Cannot reach ai-gateway.zende.sk"

Connect to GlobalProtect VPN. Test with: ping ai-gateway.zende.sk

"Token doesn't look right"

Must start with zdai_. No spaces before or after. Get a fresh one from ai-gateway.zende.sk

"Not a member of zendesk org"

Request access via Okta Zendesk Hub → Productiv. Include your GitHub username in the request.

GitHub authentication keeps failing

The wizard uses browser login automatically. Make sure you're logging in with your Zendesk GitHub account credentials.

Marketplace fails with "permission denied"

If you have SSH keys: Go to github.com/settings/keys, find your key, click Configure SSO → Authorize for "zendesk". The wizard prompts you to do this, but if you skipped it, you can fix it anytime.

Marketplace fails with "SSH host key is not in your known_hosts file"

First, try: ssh -T git@github.com (type yes when prompted). If this fails or you get "permission denied", you need to set up SSH keys — see the Manual SSH Key Setup section below.

claude command not found after install

Open a new terminal window (the PATH only updates in new shells). Or run: source ~/.zshrc

Manual SSH Key Setup (Advanced)

If the marketplace fails with SSH errors and ssh -T git@github.com doesn't work, follow these steps to set up SSH keys manually:

1. Generate SSH Key

ssh-keygen -t ed25519 -C "your_email@example.com"

Press Enter through all prompts to accept defaults.

2. Start SSH Agent and Add Key

eval "$(ssh-agent -s)" && ssh-add ~/.ssh/id_ed25519

3. Copy Public Key

cat ~/.ssh/id_ed25519.pub

This will display your public key. Copy the entire output.

4. Add Key to GitHub

  1. Go to GitHub → Settings → SSH and GPG keys
  2. Click New SSH key
  3. Paste your public key and give it a title (e.g., "Work Laptop")
  4. Click Add SSH key

5. Enable SSO for Zendesk

  1. On the same SSH keys page, find your newly added key
  2. Click Configure SSO button next to it
  3. Click Authorize for the "zendesk" organization

6. Test SSH Connection

ssh -T git@github.com

You should see: "Hi [username]! You've successfully authenticated..."
