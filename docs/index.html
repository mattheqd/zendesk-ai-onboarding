Get your Mac ready for the workshop. Half reading, half installing. You'll leave with Claude Code running on your machine, authenticated to Zendesk's AI Gateway.

Before the Workshop

Don't skip this page.

The workshop is hands-on. If your laptop isn't set up, you'll spend the session watching someone else type. Fifteen minutes the day before, forty-five if you're new to terminals. Skip nothing.

What you're installing and why:

Claude Code is not a website you log into. It's an agent that runs on your laptop, talks to your filesystem, and edits real files, authenticated to Zendesk's internal AI Gateway. Setting that up takes a few small installs. Do it once and you have these superpowers forever.

Quick Setup

Open Terminal (Cmd+Space, type "Terminal", hit Return) and paste:

bash <(curl -fsSL https://raw.githubusercontent.com/mattheqd/zendesk-ai-onboarding/main/setup-wizard.sh)

Important: Make sure GlobalProtect VPN is connected first. If ai-gateway.zende.sk is unreachable, nothing works.

1 ·

GlobalProtect VPN

Almost everything we use today lives behind the corporate network. The AI Gateway, the MCP gateway, all internal Zendesk services. If GlobalProtect is off, nothing works.

Open GlobalProtect before every session. Smoke test: run ping ai-gateway.zende.sk in your terminal. If it resolves, you're good.

GitHub access required

If you're not already in the zendesk GitHub org, you need access before the workshop. This takes ~15 minutes if you ask today, but 2 days if you wait.

Action: Ping #it-help on Slack:

"Hi! I need to be added to the Zendesk GitHub organization for the Claude Code workshop. My GitHub username is [YOUR_GITHUB_USERNAME]"
2 ·

Install Homebrew (Mac) or WSL (Windows)

Mac: Homebrew is the package manager that lets you install other tools with one line. Open Terminal (Cmd+Space, type "Terminal", hit Return) and paste:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Windows: You'll need WSL (Windows Subsystem for Linux). The setup wizard handles this, but if you want to do it manually first, search "WSL" in your start menu.

3 ·

Get your AI Gateway token

The setup wizard will open your browser for you, but here's the manual flow:

  • Open ai-gateway.zende.sk
  • Log in with your Zendesk credentials
  • Look for your API token (starts with zdai_)
  • Copy the entire token
  • Paste it into the wizard when prompted

Common Issues

Troubleshooting

"Cannot reach ai-gateway.zende.sk"

Connect to GlobalProtect VPN. Test with: ping ai-gateway.zende.sk

"Token doesn't look right"

Must start with zdai_. No spaces before or after. Get a fresh one from ai-gateway.zende.sk

"Not a member of zendesk org"

Ping #it-help on Slack. Takes ~15 min if you ask today, 2 days if you wait.

GitHub authentication keeps failing

When prompted, choose: "GitHub.com" (not Enterprise), "HTTPS" protocol, "Login with web browser" method.

claude command not found after install

Open a new terminal window (the PATH only updates in new shells). Or run: source ~/.zshrc
